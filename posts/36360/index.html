第一章 数据库基础

1.数据库(DB)

数据库的定义

  • 数据库（DataBase,DB）是长期储存在计算机内，有组织的，可共享的大量数据的集合

数据库的基本特征

  1. 数据按一定的数据模型组织，描述和储存
  2. 可为各种用户共享数据
  3. 冗余度较小
  4. 数据独立性较高
  5. 易扩展

数据库总览

  • 关系型数据库（SQL）
    • MySQL , Oracle , SQL Server , SQLite , DB2 , …
    • 关系型数据库通过外键关联来建立表与表之间的关系
  • 非关系型数据库（NOSQL）
    • Redis , MongoDB , …
    • 非关系型数据库通常指数据以对象的形式存储在数据库中，而对象之间的关系通过每个对象自 身的属性来决定

2.数据库管理系统（DBMS)

数据库管理系统是一个用于支持用户创建和维护数据库的应用程序集合,是一个计算机基础软件，一个大型复杂的软件系统

DBMS的用途

科学地组织和存储数据、高效地获取和维护数据

DBMS主要功能

  1. 数据定义功能

    1. 提供数据定义语言(Data Defination Language，DDL)
    2. 定义数据库中的数据对象

  2. 数据组织、存储和管理功能

    1. 分类组织、存储和管理各种数据，如用户数据、数据字典、索引等
    2. 确定组织数据的文件结构和存取方式
    3. 实现数据之间的联系
    4. 提供多种存取方法提高存取效率

  3. 数据操纵功能

    1. 提供数据操纵语言(Data Manipulation Language，DML)
    2. 实现对数据库的基本操作 (查询、插入、删除和修改)

  4. 数据库的事务管理和运行管理

    1. 数据库在建立、运行和维护时由DBMS统一管理和控制
    2. 保证数据的安全性、完整性、多用户对数据的并发使用以及发生故障后的系统恢复

  5. 数据库的建立和维护功能

    1. 数据库初始数据装载、转换
    2. 数据库转储，介质故障恢复
    3. 数据库的重组织
    4. 性能监视分析等

  6. 其他功能

    1. DBMS与网络中其它软件系统的通信
    2. 两个DBMS系统的数据转换
    3. 异构数据库之间的互访和互操作

    DBMS的工作模式示意图

    1）接收应用程序的数据请求和处理请求。

    2）将用户的查询数据请求（高级指令）转换成复杂的低层指令。

    3）低层指令实现对数据库的具体操作。

    4）接收数据库操作得到的查询结果。

    5）对查询结果进行处理，包括相应的格式转换。

    6）将处理结果返回给用户。

    数据库管理系统的模块组成

  • 语言编译处理程序
  • 系统运行控制程序
  • 系统建立与维护程序
  • 数据字典

3.数据库系统（DBS）

定义：在计算机系统中引入数据库的系统称为数据库系统

构成：数据库、数据库管理系统、应用程序、数据库管理员、用户

数据处理

对各种类型的数据进行收集、分类、储存、检索、加工和计算、传播等一系列活动的综合

数据管理

  • 对数据进行分类、组织、编码、存储、检索和维护
  • 数据处理的核心问题

数据库系统的结构类型

数据库系统的类型可以从不同的角度进行划分。根据用户数的多少来分类，可分为单用户和多用户；根据用途和使用范围分类为数据仓库或决策支持等类型；按数据库系统的部署位置不同可分为集中式、客户机／服务器式、分布式和并行式4种。

  1. 集中式数据库系统

数据库中的数据和数据处理集中在一台计算机上完成，其它用户可使用终端设备访问数据库，但是终端不参数据库中数据的计算与管理。此外，终端本身并不能完成任何操作,所有数据操作必须由主机完成

  1. 客户机/服务器（Client/Server，C/S）数据库系统

客户端完成本地个性化的处理，并向服务器发送请求，同时显示服务器返回的数据结果；服务器端负责处理公共的部分任务

C/S三层架构

实际应用中，出于系统安全性、稳定性和健壮性考虑，通常将数据库、应用服务器和客户端三层分开部署，称为C/S三层架构。包括：表示层、应用层和数据层。

  1. 分布式数据库系统

特点：逻辑整体性，物理分布性

分布式数据库一般应用在大型跨国或跨地区的企业或单位

  1. 并行式数据库系统

高性能：如通过将数据库在多个磁盘上分布存储,利用多个处理机对磁盘数据进行并行处理解决磁盘I/O瓶颈,提高效率

高可用性：通过数据复制增强数据库的可用性,当一个磁盘损坏时,该数据在其他磁盘上的副本仍可供使用

可扩充性：当数据库遇到性能和容量瓶颈时,可通过增加处理器和存储设备平滑扩展性能

4.数据库系统的发展及趋势

人工数据管理阶段

1946年后计算机以电子管为主要元器件,主要依靠硬件系统,工作效率低,用于计算且输入输出很少数据。

主要特点：

  1. 数据不保存
  2. 数据面向应用
  3. 数据无独立性
  4. 无专门数据管理软件

文件系统管理数据阶段

从20世纪50年代中期到60年代中期，计算机以晶体管取代了运算器和控制器中的电子管。出现了操作系统、汇编语言和一些高级语言。计算机不仅限于科学计算，还大量用于管理等，在操作系统中有专门的数据管理软件，称为文件系统。

文件系统管理数据的特点

  1. 数据长久保存
  2. 数据共享能力差
  3. 数据不能独立
  4. 数据管理功能简单

文件系统的不足

  1. 数据不共享，冗余大
  2. 数据不一致，不独立
  3. 数据文件缺乏关联

数据库系统阶段

从20世纪60年代末到80年代，CPU向超大规模集成电路发展,操作系统得到了发展，而且各种DBMS软件不断涌现，使得数据库管理技术不断发展和完善，成为计算机领域中最具影响力和发展潜力、应用范围最广、成果最显著技术之一，形成了“数据库时代”。

主要特点：

  1. 数据整体结构化
  2. 数据高共享
  3. 数据独立性高
  4. 数据统一由DBMS管理和控制

高级数据库发展阶段

从20世纪80年代以后，传统数据库技术日趋成熟，并在各个领域的应用也逐步深入，在网络技术和各行业的新需求的不断推动下，数据库技术的发展呈现出前所未有的新气象，分布数据库、数据仓库、数据集市和商业智能等新技术纷纷涌现。

分布式数据技术

主要特点：

  1. 物理分布性
  2. 逻辑整体性
  3. 站点自治性
  4. 站点协作性
  5. 数据分布透明性
  6. 存在适当的数据冗余
  7. 事务管理的分布性

分布式数据库系统兼顾集中管理和分布处理两项任务

面向对象数据库技术

主要特点

  1. 对象数据模型能完整地描述现实世界中的数据结构，表达数据间的嵌套，递归等关系
  2. 具有面向对象技术的封装性（数据与操作定义一起）和继承性（继承数据结构和操作）特点，提高软件的可用性

5.数据模型

模型：对现实世界中某一事物特征的模拟和抽象

数据模型的分类

  1. 概念模型：也称信息模型，它是按用户的观点来对数据和信息建模，用于数据库设计
  2. 逻辑模型：按计算机系统的观点对数据建模，描述数据的整体逻辑结构，主要用于DBMS实现，包括关系模型、层次模型、网状模型、面向对象模型等
  3. 物理模型：是对数据最底层的抽象，描述数据在系统内部的表示方式和存取方法，在磁盘或磁带上的存储方式和存取方法

信息世界中的基本概念

  1. 实体：实体是现实世界中独立存在的事物，如：人，汽车，动物…
  2. 属性：实体所具有的某一特征，如：学号，姓名，性别…
  3. 域：属性的取值范围，如：年龄的域是整数，生日的域是日期
  4. 联系：实体间的相互关系，按照实体参与联系可分为三类：一对一，一对多，多对多

6.数据库的模式结构

三级模式结构

三级模式：模式，内模式，外模式

内模式：一个

也称存储模式或物理模式

  • 数据物理结构和存储方式的描述
  • 数据在数据内部的表示方式

模式：一个

模式地位：数据库系统模式的中间层

  • 与数据库的物理存储细节和硬件无关
  • 与具体的应用程序、开发工具等无关

外模式：多个

也称子模式或用户模式

一个数据库可以有多个外模式，反映了不同的用户需求，看待数据的方式，对数据保密的要求

一个应用程序只能拥有一个外模式

二级映像

外模式/模式映像保证了数据的逻辑独立性

  • 当模式改变时，数据库管理员修改有关的外模式／模式映象，使外模式保持不变
  • 应用程序是依据数据的外模式编写的，从而应用程序不必修改，保证了数据与程序的逻辑独立性，简称数据的逻辑独立性

模式／内模式映象保证了数据的物理独立性

  • 数据库中的模式/内模式映像是唯一的
  • 当数据库的存储结构发生改变时，只需要修改模式映像即可

第二章 关系数据库基础

1.基本概念

域：一组具有相同数据类型的值的集合，域决定了属性的取值范围

笛卡尔积：是域上的集合运算

笛卡尔积的基数为 2×2×3=12

关系：D1×D2×…×Dn的子集叫作在域D1，D2，…，Dn上的关系，表示为 R（D1，D2，…，Dn）

关系，属性，元组

  1. 关系是笛卡尔积的一个有意义的自己
  2. 每一列称为属性，对应一个域，并起一个名称
  3. 每一行为一个元组
  4. 关系是元组的集合

2.关于码的几个术语

码：码是我们一个足够能区分关系元组的==属性==集合

超码：若关系中的某一属性组的值能唯一地标识一个元组，则称该属性组为超码，超码中可能包含一些无关紧要的属性

候选码：==候选码是最小超码==，其任意真子集都不能为超码

主码：从候选码选定一个用于标识的==元组==，称为主码

主属性：候选码中各属性称为主属性

非主属性：不包含在任何候选码中的属性

全码：若关系模式中的所有属性都是这个关系模式的候选码，则称为全码，所有属性共同确定唯一一个元组

3.关系模型的完整性

关系的三类完整性约束

关系模型的完整性规则是对关系的某种约束条件

三类完整性约束：

  1. 实体完整性
  2. 参照完整性
  3. 用户定义完整性

实体完整性和参照完整性是用户必须满足的约束条件，成为两个关系的不变性，由DBMS自动支持

用户定义完整性是应用领域需要遵守的约束条件

实体完整性

若属性A是基本关系的主属性，则属性A不能为空值(NULL)

关系模型中以主码作为唯一标识，==主码既不能为空，也不能重复==

参照完整性

它表示的是两个关系之间的联系，是表与表之间的引用。对应的就是外键，外键约束实现参照完整性

用户定义的完整性

即规定属性的取值范围

第四章 数据库、表和数据操作

1.结构化查询语句分类

名称 解释 命令
DDL（数据定义语言） 定义和管理数据对象，如数据库，数据表等。 CREAT,DROP,ALTER
DML（数据操作语言） 用于操作数据库对象中所包含的数据 INSERT,DELETE,UPDATE
DQL（数据查询语言） 查询数据库数据 SELECT
DCL（数据控制语言） 用于管理数据库的语言，包括管理权相及数据更改 GRANT,RECOKE,DENY

2.数据库操作

命令行操作数据库

创建数据库：create database [if not exists] 数据库名

删除数据库：drop database [if exists] 数据库名

查看数据库：show databases

使用数据库：use 数据库名

创建数据表

create table [if not exists] `表名`(
	`字段名1` 列类型 [属性] [索引] [注释],
    # ...
)
create table 学生(
	学号 char(10) primary key,
	姓名 char(5) not null,
    # ...
);

删除数据

delete from `表名` 
where <条件表达式>

修改数据

update `表名` set `列名`=`修改后的值`
where <条件表达式>

增加数据

# 字段名可以部分省略，但添加的值必须与字段名对应，且数量一致
insert into `表名`(字段1，字段2，...，字段n)
values (值1，值2，...，值n)

查询数据

select [all | distinct] *
from `表名`
where ...     ------ 指定结果满足满足的条件
group by ...  ------ 指定结果按照哪几个字段来分组
having ...    ------ 过滤分组要满足的次要条件
order by ...  ------ 指定结果按一定的方式排序，默认升序，desc：降序，缺省值为降序

模糊查询

like 运算符用于字符串匹配

通配符

  • %（百分号）：代表任意个任意字符，例如a%b
  • __（下划线）：代表一个任意字符，例如a_b
  • []（方括号)：指定范围中的单个字符
# 查询所有书名以“数据库”开头的图书
select * from 图书
where 书名 like '数据库%'

# 查询所有书名含有“计算机”的图书信息
select * from 图书
where 书名 like '%计算机%'

# 查询以"DB_"开头，且倒数第3个字符为 i的图书信息
select * from 图书
where 书名 like 'DB/_%i__'escape'\'

集合查询

并：UNION

交：INTERSECT

差：EXCEPT

参加集合操作的各查询结果的列数必须相同；对应项的数据类型也必须相同

第五章 索引与视图

索引

1.概念

==索引==是关系中一列/几列值及相应的指向关系中标识，这些值的数据页的逻辑指针（地址）的==列表==

  • 索引是键值（关键字）与其数据行地址的对应表
  • 索引是有序的

2.特点

  1. 极大地提高数据查询速度
  2. 通过创建唯一索引，可以==保证数据库中各行数据的唯一性==
  3. 建在外键的索引可以==加速==多表之间的连接，有益实现数据的参照完整性
  4. 查询涉及到分组和排序时，也可==显著减少分组和排序时间==
  5. 通过使用索引可以在查询过程中使用优化隐藏器，==提高系统性能==。索引提高检索性能具有一定代价
    • 存储空间中除了存放数据表，还需要存放索引的空间
    • 对数据表插入、修改和删除操作时，相应的索引也需要动态维护更新，消耗系统资源

3.类型

  • 聚簇索引 唯一

    聚集索引确定表中数据的物理顺序，它要求数据行按键值在表内有序存储，从而使得数据表的物理顺序与索引顺序一致

    • 提供高度唯一性
    • 可用于经常使用的查询
    • 可用于范围查询

  • 非聚簇索引 多个

  • 唯一索引

  • 复合索引

    • 建立在多个列上的索引称为复合索引
    • 复合索引中，系统==按照索引列出现的先后顺序==对索引项排序

  • 其他索引

4.索引操作

索引的创建

create [unique] [clustered]
index <索引名> on `表名`(列名)

索引的查看

exec sp_helpindex <表名>
dbcc show_statistiics(表名，索引名)

索引的删除

drop index <索引名> on <表名或视图名>
drop index <表名或视图名>.<索引名>

视图

1. 概念

2.视图的基本操作

视图的创建

# 格式
create view <视图名> 
as
	子查询
	with check option

# 示例
create view professor
as
	select 工号,系部编号, 教师姓名,性别,年龄, 职称
	from 教师
	where 职称='教授'
	with check option

with check option 作用

  1. 对于update,有with check option，要保证update后，数据要被视图查询出来
  2. 对于delete,有无with check option都一样
  3. 对于insert,有with check option，要保证insert后，数据要被视图查询出来
  4. 对于没有where 子句的视图，使用with check option是多余的

视图的修改

alter view <视图名>
as <子查询>

视图的删除

drop view <视图名>
